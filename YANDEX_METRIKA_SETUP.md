# 📊 Настройка Яндекс.Метрики

## Шаг 1: Создать счетчик

1. Перейдите на https://metrika.yandex.ru
2. Войдите через Яндекс ID
3. Нажмите "Добавить счетчик"
4. Заполните данные:
   - **Название:** Descro
   - **Адрес сайта:** https://descro-production.up.railway.app
   - **Часовой пояс:** UTC+3 (Москва)
5. Нажмите "Создать счетчик"

## Шаг 2: Получить ID счетчика

После создания вы увидите **номер счетчика** (например: 98765432)

## Шаг 3: Добавить ID в переменные окружения

### Локально (.env.local):
```env
NEXT_PUBLIC_YANDEX_METRIKA_ID=98765432
```

### Railway:
1. Откройте проект в Railway
2. Перейдите в Variables
3. Добавьте переменную:
   - Name: `NEXT_PUBLIC_YANDEX_METRIKA_ID`
   - Value: `98765432` (ваш ID)
4. Сохраните и передеплойте

## Шаг 4: Проверить работу

1. Откройте сайт: https://descro-production.up.railway.app
2. Откройте DevTools → Network
3. Найдите запрос к `mc.yandex.ru/watch/`
4. Если запрос есть - метрика работает ✅

Или:
1. Зайдите в Яндекс.Метрику
2. Откройте свой счетчик
3. Перейдите в "Отчеты" → "Стандартные отчеты" → "Посещаемость"
4. Откройте сайт в новой вкладке
5. Через 10-30 секунд должен появиться визит

## Что отслеживается

✅ **Автоматически:**
- Просмотры страниц
- Клики по ссылкам
- Время на сайте
- Источники трафика
- География посетителей
- Устройства и браузеры
- Карта кликов (clickmap)
- Вебвизор (запись сессий)

✅ **E-commerce события:**
- Покупка PRO подписки
- Отмена подписки
- Возвраты

## Полезные отчеты

### 1. Воронка конверсии
Отчеты → Стандартные отчеты → Конверсии → Цели

Настройте цели:
- **Регистрация:** URL содержит `/dashboard`
- **Генерация:** JavaScript событие `generate`
- **Покупка PRO:** URL содержит `/payment/success`

### 2. Карта кликов
Отчеты → Поведение → Карта кликов

Показывает, куда пользователи кликают чаще всего.

### 3. Вебвизор
Отчеты → Поведение → Вебвизор

Записи реальных сессий пользователей (анонимно).

### 4. Источники трафика
Отчеты → Источники → Источники, сводка

Откуда приходят пользователи:
- Прямые заходы
- Поисковые системы (Google, Яндекс)
- Социальные сети
- Реферальные ссылки

## Отправка кастомных событий

Если хотите отслеживать дополнительные события:

```typescript
// В любом клиентском компоненте
declare global {
  interface Window {
    ym: any
  }
}

// Отправить событие
if (typeof window !== 'undefined' && window.ym) {
  window.ym(98765432, 'reachGoal', 'generate_description')
}
```

### Примеры событий:

```typescript
// Генерация описания
window.ym(METRIKA_ID, 'reachGoal', 'generate')

// Покупка PRO
window.ym(METRIKA_ID, 'reachGoal', 'purchase', {
  order_price: 199,
  currency: 'RUB'
})

// Регистрация
window.ym(METRIKA_ID, 'reachGoal', 'register')

// Клик по кнопке
window.ym(METRIKA_ID, 'reachGoal', 'click_pro_button')
```

## E-commerce отслеживание

Для отслеживания покупок добавьте в webhook:

```typescript
// app/api/payment/webhook/route.ts
if (body.event === 'payment.succeeded') {
  // ... активация PRO ...
  
  // Отправить событие в dataLayer для Метрики
  console.log('E-commerce event:', {
    ecommerce: {
      purchase: {
        actionField: {
          id: payment.id,
          revenue: 199
        },
        products: [{
          id: 'pro-subscription',
          name: 'PRO подписка',
          price: 199,
          quantity: 1
        }]
      }
    }
  })
}
```

## Настройка целей в Метрике

1. Откройте счетчик в Яндекс.Метрике
2. Настройки → Цели → Добавить цель

### Цель 1: Регистрация
- Название: Регистрация
- Тип: URL
- Условие: содержит `/dashboard`

### Цель 2: Генерация
- Название: Генерация описания
- Тип: JavaScript событие
- Идентификатор: `generate`

### Цель 3: Покупка PRO
- Название: Покупка PRO
- Тип: URL
- Условие: содержит `/payment/success`

### Цель 4: Возврат
- Название: Запрос возврата
- Тип: JavaScript событие
- Идентификатор: `refund_request`
